## The Crisis Nobody Is Solving

**70% of published scientific findings cannot be replicated.**

That number is not a typo. It is the reproducibility crisis — and it is quietly costing the world billions of dollars in wasted follow-up research, corrupting medical decisions, and eroding the public trust in science itself.

The problem is not that scientists are dishonest. The problem is that **peer review is broken by design.**

Reviewers are overworked. They check one paper at a time. They cannot cross-reference figures across 50 pages. They cannot detect when p-values have been massaged. They cannot run forensic analysis on every image in a manuscript.

**A machine can. That is what SciPeerAI is.**

## What It Does — 14 Modules

<table>
<tr>
<td width="50%">

### ✅ Module 01 — Statistical Audit Engine
Detects p-hacking by identifying suspicious clusters near the 0.05 threshold. Flags underpowered sample sizes and suspiciously round numbers.

### ✅ Module 02 — Figure Forensics Pipeline
Extracts every image from a PDF and runs three-layer forensic analysis: perceptual hashing for duplicates, Error Level Analysis for JPEG artifacts, brightness uniformity scoring.

### ✅ Module 03 — Methodology Logic Checker
Rule-based and LLM-powered reasoning. Detects causation claims without RCT, missing control groups, timeframe mismatches.

### ✅ Module 04 — Citation Integrity Analyzer
Detects self-citation abuse, unsupported broad claims, low citation density, and citation cartel signals.

### ✅ Module 05 — Reproducibility Scanner
Scores paper across 8 dimensions: code, data, software versions, preregistration, ethics, IRB approval.

### ✅ Module 06 — Novelty Scorer
Structural language signals combined with live Semantic Scholar search. Detects incremental vs genuinely novel work.

### ✅ Module 07 — GRIM Test
Detects mathematically impossible means. If n=15 and reported mean is 3.47, the math proves the data cannot be real.

</td>
<td width="50%">

### ✅ Module 08 — SPRITE Test
Detects impossible score distributions. Reconstructs all possible integer combinations and checks if reported SD is achievable.

### ✅ Module 09 — Granularity Analyzer
Applies Benford's Law to numeric data. Real measurements follow predictable digit distributions — fabricated data does not.

### ✅ Module 10 — P-Curve Analyzer
Detects publication bias by analyzing the shape of p-value distributions. Genuine effects produce right-skewed p-curves.

### ✅ Module 11 — Effect Size Validator
Validates Cohen's d, correlation coefficients, and odds ratios against sample sizes. Catches inflated effects and underpowered claims.

### ✅ Module 12 — Retraction Checker
Live CrossRef API integration. Checks every cited DOI against retraction records in real time.

### ✅ Module 13 — Citation Cartel Detector
Graph-based network analysis. Detects coordinated mutual citation rings that artificially inflate impact metrics.

### ✅ Module 14 — LLM Paper Detector
Detects AI-generated papers using burstiness scoring, type-token ratio, sentence uniformity, and LLM phrase detection.

</td>
</tr>
</table>

## Quick Start

```bash
git clone https://github.com/Abu-Sameer-66/SciPeerAI.git
cd SciPeerAI
conda create -n scipeerai python=3.10 -y
conda activate scipeerai
pip install -r requirements.txt
python main.py
```

Local API → `http://localhost:8000`  
Live API  → `https://web-production-f526d.up.railway.app`  
Live UI   → `https://scipeerai-ui.vercel.app`

## API Usage

```bash
# statistical fraud detection
curl -X POST "https://web-production-f526d.up.railway.app/api/v1/analyze/statistics" \
  -H "Content-Type: application/json" \
  -d '{"text": "n=12 participants. p=0.048. Secondary also p=0.049."}'

# GRIM test
curl -X POST "https://web-production-f526d.up.railway.app/api/v1/analyze/grim" \
  -H "Content-Type: application/json" \
  -d '{"text": "n=15, mean=3.47, SD=1.2"}'

# retraction check
curl -X POST "https://web-production-f526d.up.railway.app/api/v1/analyze/retraction" \
  -H "Content-Type: application/json" \
  -d '{"text": "Wakefield et al. doi:10.1016/S0140-6736(97)11096-0"}'

# LLM paper detection
curl -X POST "https://web-production-f526d.up.railway.app/api/v1/analyze/llm" \
  -H "Content-Type: application/json" \
  -d '{"text": "paste paper text here"}'

# figure forensics — PDF upload
curl -X POST "https://web-production-f526d.up.railway.app/api/v1/analyze/figures" \
  -F "file=@paper.pdf"
```
